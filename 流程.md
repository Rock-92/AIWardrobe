# AIWardrobe 当前流程说明

更新时间：2026-06-05

本文档记录当前 AI 智能衣柜原型已经实现到的流程和模块状态。当前项目仍是本地 Web 原型，入口为 `index.html`、`app.js`、`styles.css` 和 `server.js`，默认服务端口为 `4173`。

## 1. 当前主流程

1. 用户选择或新建用户。
2. 用户补充需求信息：身高、体型、年龄、天气、场合、心情、推荐边界。
3. 用户在对话框输入本次穿搭需求。
4. 前端保存用户消息，并调用 `runRecommendation()`。
5. 后端调用千问完成输入审核、需求解析和偏好增量抽取。
6. 前端对解析结果做机械审核，避免 prompt injection、跨用户数据、隐私风险和错误偏好写入。
7. 如果审核通过，合并本轮偏好到用户长期记忆。
8. 使用 `retrieval_query` 检索样例数据，得到相关 `sample-*` 参考样例。
9. 将用户资料、需求解析、历史偏好、当前衣柜、检索样例和生成规则组成 prompt。
10. 调用千问生成三套穿搭方案。
11. 前端校验衣物编号、来源和规则一致性。
12. 输出自然语言推荐，并在右侧显示检索证据、校验结果和模型输入摘要。
13. 用户可选择某一套方案，调用万相生成穿搭效果图。

## 2. 用户与需求信息

用户信息包括：

- 用户名称
- 性别
- 偏好风格
- 偏好单品
- 偏好颜色
- 历史偏好记忆
- 当前衣柜
- 对话历史

需求区当前支持：

- 身高
- 体型
- 年龄
- 天气
- 场合
- 心情
- 推荐边界：衣柜优先 / 可加灵感

这些信息会进入需求解析、RAG 检索、推荐生成和穿搭效果图 prompt。

## 3. 衣柜管理流程

当前衣柜支持：

- 搜索衣物编号或名称
- 进入衣柜编辑模式
- 添加衣物
- 编辑已有衣物
- 删除衣物
- 本地上传衣物图片
- 根据衣物名称生成本地衣物示意图

衣物条目保存字段：

- 名称
- 描述
- 图片
- 图片来源
- 生成图片本地路径

添加衣物规则：

- 只有名称：调用模型生成衣物描述和示意图。
- 只有图片：调用模型识别衣物特征并生成描述。
- 名称和图片都有：优先保留用户图片，结合名称生成描述。

删除衣物规则：

- 用户点击删除按钮后，会弹出确认框：`#编号-名称`。
- 只有用户确认后才删除。
- 如果该衣物图片是模型生成并保存在 `GeneratedWardrobeImages` 中，会同步删除本地生成图。
- 如果图片来自用户本地上传，不额外删除用户原始文件。

## 4. 衣物图片生成规则

根据衣物名称生成示意图时，prompt 当前要求：

- 只生成单件衣物商品图。
- 背景必须是纯白色 `#FFFFFF`。
- 不允许灰墙、地面、桌面、阴影、渐变、纹理、环境物体或装饰。
- 不允许真人模特、文字水印、品牌标志。
- 鞋、包、裤子、上衣、外套都只展示单品本身。

生成图会保存到本地 `GeneratedWardrobeImages`，并在衣柜条目中记录路径。

## 5. 历史偏好生成与展示

偏好记忆来自模型解析出的 `preference_delta`，包括：

- `likes`
- `dislikes`
- `contextual_preferences`
- `do_not_store`
- `summary`

写入规则：

- 只有通过安全审核和机械审核的输入才允许写入偏好记忆。
- 偏好按关键词本身去重，不再按 `type + value` 去重。
- 如果新偏好和旧偏好关键词相同，只保留置信度更高的一条。
- 场景化偏好也会去重并保留最近记录。

展示规则：

- 右侧“历史偏好”优先展示模型提取出的动态关键词。
- 固定词表只作为兜底。
- 只展示出现次数大于等于 2 次的词。
- 标签只展示词本身，不再显示 `×次数`。

删除规则：

- 右侧“历史偏好”和用户信息弹窗里的“历史对话提取”都支持删除。
- 删除后会从 `likes`、`dislikes`、`contextual_preferences` 中真正移除。
- 不再写入隐藏 buffer 或 `suppressed` 列表。
- 如果展示词来自长词片段，例如展示 `正式` 但真实记忆是 `正式但不失轻松`，删除 `正式` 会一并移除对应长词。

## 6. RAG 与检索证据

当前 RAG 主要检索穿搭样例数据：

- 使用需求解析得到的 `retrieval_query`。
- 检索结果以 `sample-*` 形式进入生成 prompt。
- 右侧“匹配样例”只在生成搭配后显示。
- 样例展示为编号加风格标签，并支持预览图片。

RAG 样例只作为风格、配色、场景参考，不能被当作用户已有衣物。

## 7. 防幻觉与校验

当前防幻觉策略包括：

- 衣柜模式优先使用当前用户衣柜中的真实编号。
- 灵感单品需要与衣柜单品区分。
- 前端会校验模型输出的衣物编号。
- 检索证据、模型输入摘要和校验结果会展示在右侧。
- API 失败时会使用本地规则兜底，避免流程中断。

## 8. 穿搭效果图生成

用户生成三套推荐后，可以点击“看看搭配上身”，选择方案生成效果图。

当前效果图生成规则：

- 使用万相模型。
- 优先参考用户衣柜中的衣物图片。
- 没有参考图的单品按名称、描述、方案风格和整体配色生成。
- 必须展示搭配中的上半身、下半身以及鞋包等配件。
- 只展示头部以下的身体部分。
- 画面从脖颈或肩部以下开始裁切。
- 不要露脸、不要五官、不要完整头部。
- 不生成用户本人，不添加品牌标志或文字水印。
- 生成图标注为虚拟模特效果图，不代表真实试穿结果。

## 9. 界面布局状态

当前界面为三栏布局：

- 左栏：用户、需求、衣柜。
- 中栏：穿搭对话、衣物添加/编辑、效果图生成。
- 右栏：检索证据、历史偏好、匹配样例、校验结果、模型输入摘要。

布局规则：

- 普通状态保持一屏高度。
- 有图片或衣物编辑时，整体高度跟随中间区域。
- 左右栏跟随中间栏高度伸展，长内容在栏内滚动。
- 衣柜标题行已优化，添加/完成按钮不换行。
